# Regra MULTISOFT — UX/UI obrigatório (baseline)

- **Clareza acima de estética**.
- **Consistência**: mesmas ações → mesmos padrões (botões, labels, validações).
- **Prevenção de erros**: validar antes, confirmar ações irreversíveis, defaults seguros.
- **Feedback imediato**: loading/sucesso/erro sempre visíveis.
- **Estados obrigatórios**: vazio, carregando, sucesso, erro, sem permissão.
- **Microcopy humana**: nada de mensagens técnicas para usuário final.
- **Acessibilidade**: contraste, teclado, foco visível, aria-labels quando necessário.
- **Mobile-first/responsivo**: testar e garantir usabilidade em telas pequenas.
